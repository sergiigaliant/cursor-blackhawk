Основні команди в Git

  • git config --global user.name blabla - Встановлює ім'я, яке буде відображатися в полі автора у виконаних вами коммітів
  • git config --global user.email blabal@bla - Встановлює адресу електронної пошти, яка буде відображатися в полі автора у виконаних вами коммітів
  • git init - Створює новий локальний репозиторій з заданим ім'ям
  • git status - Перераховує всі нові або змінені файли, які потребують фіксації
  • git add "filename" - Добавити вказаний файл в індекс (staging area)
  • git add . - Добавити усі файли в індекс (staging area)
  • git commit -m "commit text" - Фіксує проіндексовані зміни і зберігає їх в історію версій
  • git log - Історія коммітів для поточної гілки (branch)
  • git checkout - Перемикається на обрану гілку і оновлює робочу директорію до її стану
  • git branch - Дозволяє показувати список гілок, створювати нові, перейменовувати та видаляти
  • git push - Завантажує всі зміни локальної гілки в віддалений репозиторій
  • git pull - Завантажує історію з віддаленого сховища та об'єднує її з локальною. git pull = git fetch + git merge
  • git merge - Вносить зміни з гілки віддаленого сховища в поточну гілку локального сховища
  • git fetch - Викачує всю історію з віддаленого сховища
  • git clone - Клонування віддаленого репозиторія
  • Файл .gitignore служить для збереження в ньому файлів і папок, які необхідно приховати від системи контролю версій git.

Advanced

  • git revert - Відміняє коміт (виконує повну протилежність, те що було знищено вертає, те що було добавлено знищує)
  • git reset - Відміняє всі комміти після заданого залишаючи всі зміни в робочій директорії
  • git rebase - Перезаписати історію коммітів при злитті гілок
  • git cherry-pick - Для перенесення коммітів з однієї гілки в іншу
  • git stash - Тимчасово зберігає всі незафіксовані зміни відслідковуваних файлів
